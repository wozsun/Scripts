# Windows 脚本

这个文件夹用于存放 Windows 平台可用的 PowerShell 7 脚本。

`common.psm1` 是本目录脚本共用的工具模块，提供动态进度条、延迟警告输出、路径输入解析、同目录专属临时目录管理等公共能力。进度条默认在百分比变化时刷新；如果百分比不变但距离上次刷新超过 `1000ms`，也会刷新当前计数，可修改 `common.psm1` 顶部 `$ProgressBarTimedRefreshMilliseconds` 调整。直接运行本目录脚本时请保留 `common.psm1` 和脚本在同一目录下；需要单文件分享时使用仓库 `Share` 目录中的共享脚本。

## Convert-OfficeFiles.ps1

`Convert-OfficeFiles.ps1` 用于调用本机 Office 原生应用，将指定文件或文件夹下的 `.doc`、`.xls`、`.ppt` 转换为 `.docx`、`.xlsx`、`.pptx`。

脚本需要 PowerShell 7 和本机已安装的桌面版 Office。脚本默认保留原始文件；如果目标新格式文件已经存在，会提示并跳过，不会覆盖。脚本只创建新格式文件，不需要预览确认。

### 参数说明

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `Path` | 无 | 一个或多个文件或文件夹绝对路径；文件会直接转换，文件夹会递归扫描；未输入路径时会引导交互输入。 |
| `-s` | 关闭 | 包含隐藏文件和隐藏文件夹。默认只扫描未隐藏项。 |
| `-h` | 关闭 | 显示帮助信息。 |

### 使用示例

查看帮助信息：

```powershell
pwsh -File .\Convert-OfficeFiles.ps1 -h
```

转换指定文件夹下的旧格式 Office 文件：

```powershell
pwsh -File .\Convert-OfficeFiles.ps1 "C:\Path\Folder"
```

直接转换一个或多个旧格式 Office 文件：

```powershell
pwsh -File .\Convert-OfficeFiles.ps1 "C:\Path\A.doc" "C:\Path\B.xls" "C:\Path\C.ppt"
```

包含隐藏文件和隐藏文件夹：

```powershell
pwsh -File .\Convert-OfficeFiles.ps1 -s "C:\Path\Folder"
```

### 注意事项

- 路径必须是 Windows 文件或文件夹绝对路径，例如 `C:\Users\Name\Documents` 或 `C:\Users\Name\Documents\A.doc`；命令行传参时，路径包含空格、括号等 PowerShell 特殊字符，请使用英文引号包裹路径；交互输入时可以粘贴带首尾英文引号的路径。
- 脚本使用 Word、Excel、PowerPoint 的原生 COM 自动化能力，需要本机安装桌面版 Office。
- 脚本默认保留原始 `.doc`、`.xls`、`.ppt` 文件。
- 脚本只创建新格式文件，不删除、移动或覆盖原始文件，因此不会展示预览菜单。
- 如果目标 `.docx`、`.xlsx`、`.pptx` 已存在，脚本会跳过，不覆盖。
- `.xls` 转 `.xlsx` 不会保留宏；如果需要保留宏，应另行转换为 `.xlsm`。
- 有密码、损坏、受保护视图、外部链接或需要人工确认的文件可能转换失败。
- 每个文件单独转换，单个文件失败时记录错误并继续处理后续文件。
- 只有存在待转换文件时，脚本才会在输入目录下创建 `.convert-officefiles-tmp`；如果该临时目录已存在，会先清空再复用，不会阻塞转换。
- 转换时会使用输入目录下本轮实际创建的临时文件夹保存临时文件；直接传入文件时，临时文件夹位于该文件所在目录。全部目标文件已存在时只输出跳过，不会创建临时目录。全部文件转换完成并退出 Office 后，再统一移动到目标位置。
- 统一移动阶段会显示百分比进度条和文件数量，每个文件移动前默认等待 `1000ms`；如需调整，可修改脚本顶部的 `$FileMoveDelayMilliseconds` 配置项。
- 移动完成后脚本会自动清理本次运行创建的临时文件夹。
- 转换过程会启动 Office 应用实例，转换完成后脚本会自动退出这些实例。

### 执行流程

脚本会按输入目录或直接文件所在目录分组，先扫描或检查输入文件并输出转换计划摘要；存在待转换文件时，再创建本轮专属临时目录，然后把所有目标不存在的旧格式文件转换到该临时文件夹。全部转换完成后，脚本会退出 Office 应用实例，再统一移动临时文件到目标位置。目标文件已存在时会逐条提示并跳过。

## Expand-ArchiveFiles.ps1

`Expand-ArchiveFiles.ps1` 用于批量解压 `.zip` / `.rar` / `.7z` 压缩包。输入路径可以是压缩包，也可以是文件夹；文件夹会递归扫描其中的 `.zip` / `.rar` / `.7z` 文件。脚本会按压缩包真实路径自动去重，并把每个压缩包解压到自身所在目录下的同名文件夹。

脚本需要 PowerShell 7。`.zip` 使用 .NET 原生能力解压，并对旧中文 ZIP 文件名编码做 GBK 回退；`.rar` / `.7z` 优先使用 Windows 内置 `tar.exe`，失败、缺失或不可用时回退到 7-Zip 或 WinRAR；`.rar` 还会尝试 UnRAR。如果目标文件夹名已存在，会自动追加序号，例如 `Archive (2)`，不会覆盖已有文件夹或文件。

### 参数说明

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `Path` | 无 | 一个或多个 `.zip` / `.rar` / `.7z` 压缩包或文件夹绝对路径；文件夹会递归扫描；未输入路径时会引导交互输入。 |
| `-s` | 关闭 | 扫描目录时包含隐藏文件和隐藏文件夹。默认只扫描未隐藏项。 |
| `-yes` | 关闭 | 跳过解压预览和删除确认菜单，解压后默认删除成功解压的原压缩包。 |
| `-h` | 关闭 | 显示帮助信息。 |

### 使用示例

查看帮助信息：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 -h
```

解压单个压缩包：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 "C:\Path\Archive.zip"
```

解压单个 RAR 压缩包：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 "C:\Path\Archive.rar"
```

解压单个 7z 压缩包：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 "C:\Path\Archive.7z"
```

扫描并解压文件夹下的 `.zip` / `.rar` / `.7z` 压缩包：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 "C:\Path\Folder"
```

同时输入压缩包和文件夹，脚本会自动去重：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 "C:\Path\Archive.zip" "C:\Path\Folder"
```

包含隐藏文件和隐藏文件夹：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 -s "C:\Path\Folder"
```

跳过解压预览和删除确认，执行默认解压并删除成功解压的原压缩包：

```powershell
pwsh -File .\Expand-ArchiveFiles.ps1 -yes "C:\Path\Folder"
```

### 注意事项

- 路径必须是 Windows 文件或文件夹绝对路径；命令行传参时，路径包含空格、括号等 PowerShell 特殊字符，请使用英文引号包裹路径；交互输入可分行，也可在同一行用空格或英文分号分隔，路径含空格或英文分号时，请使用英文引号包裹路径。
- 当前支持 `.zip`、`.rar` 和 `.7z` 压缩包；其他格式会被跳过并输出原因。
- 旧压缩软件生成的中文 ZIP 若没有 UTF-8 文件名标记，脚本会自动尝试 GBK 文件名编码，避免解压后文件名乱码或因乱码路径导致失败。
- `.rar` / `.7z` 优先使用 Windows 内置 `tar.exe`；如果失败或不可用，脚本会依次尝试 7-Zip、WinRAR；`.rar` 还会尝试 UnRAR / RAR。
- 多个输入路径可以重复或互相包含，脚本会按压缩包真实路径自动去重，避免同一压缩包被重复解压。
- 默认解压目标是压缩包同目录下的同名文件夹；如果同名文件夹或文件已存在，会自动使用带序号的新文件夹名，不会覆盖。
- 除 `-yes` 外，解压前会先显示详细预览；可选择默认解压、手动解压或退出脚本。
- 解压完成后，脚本会列出成功解压的原压缩包并询问是否删除；选择删除时会再次检查解压目录和原压缩包是否存在，然后只删除检查通过的原压缩包。
- `-yes` 会跳过解压预览和删除确认菜单，解压完成后默认删除成功解压的原压缩包；默认删除前仍会显示危险操作倒计时，倒计时期间可按 `Enter` 取消删除。
- 解压时会先写入压缩包所在目录下的 `.expand-archivefiles-tmp` 临时文件夹；如果该临时目录已存在，会先清空再复用，不会阻塞解压。成功后再移动为最终文件夹；单个压缩包失败时会清理本轮临时文件并继续处理后续压缩包。
- 扫描、解压或移动失败时会提示原因并继续处理后续压缩包。

### 执行流程

脚本会先解析输入路径，递归扫描目录并按压缩包真实路径去重，再为每个压缩包计算不会覆盖已有项目的目标文件夹。确认执行后，脚本逐个压缩包解压到同目录下的专属临时文件夹，成功后移动为最终目标文件夹。解压阶段完成后，脚本会根据成功解压列表进入原压缩包删除阶段。最后输出解压和删除数量。

## Remove-DuplicateFiles.ps1

`Remove-DuplicateFiles.ps1` 用于查找内容完全相同的文件，并删除重复文件。

脚本需要 PowerShell 7。无路径且未指定 `-a` 或 `-c` 时，会先显示扫描模式菜单；交互模式每轮流程完成后会返回菜单，命令行带路径运行时执行一次后退出。默认不扫描隐藏项；传入 `-s` 可包含隐藏文件和隐藏文件夹。

### 模式

- 单目录模式：不传入 `-a` 或 `-c` 时启用；多个目录会先全部扫描，再逐个目录操作。
- 多目录合并模式：传入 `-a` 时启用；多个目录视作一个大目录。
- 参考目录模式：传入 `-c` 时启用；第一个目录为参考目录，只删除后续目标目录中的重复文件。

### 参数说明

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `Path` | 无 | 一个或多个文件夹绝对路径；未输入路径时会引导交互输入；未指定模式时会先选择扫描模式。 |
| `-a` | 关闭 | 多目录合并模式，把所有输入目录视作一个虚拟大目录扫描。 |
| `-c` | 关闭 | 参考目录模式，第一个目录为参考目录，其余目录为目标目录。 |
| `-s` | 关闭 | 包含隐藏文件和隐藏文件夹。默认只扫描未隐藏项。 |
| `-yes` | 关闭 | 跳过详细预览和菜单，输出汇总后等待 10 秒，再执行默认删除；倒计时期间可按 `Enter` 取消。 |
| `-h` | 关闭 | 显示帮助信息。 |

### 使用示例

单目录模式：

```powershell
pwsh -File .\Remove-DuplicateFiles.ps1 "C:\Path\Folder"
```

多个目录先全部扫描，再逐个目录操作：

```powershell
pwsh -File .\Remove-DuplicateFiles.ps1 "C:\Path\A" "C:\Path\B"
```

包含隐藏文件和隐藏文件夹：

```powershell
pwsh -File .\Remove-DuplicateFiles.ps1 -s "C:\Path\Folder"
```

跳过详细预览，输出汇总后执行默认删除：

```powershell
pwsh -File .\Remove-DuplicateFiles.ps1 -yes "C:\Path\Folder"
```

多目录合并模式：

```powershell
pwsh -File .\Remove-DuplicateFiles.ps1 -a "C:\Path\A" "C:\Path\B" "C:\Path\C"
```

参考目录模式：

```powershell
pwsh -File .\Remove-DuplicateFiles.ps1 -c "C:\Path\Reference" "C:\Path\TargetA" "C:\Path\TargetB"
```

### 规则与安全

- 路径必须是 Windows 文件夹绝对路径；命令行传参时，路径包含空格、括号等 PowerShell 特殊字符，请使用英文引号包裹路径；交互输入可分行，也可在同一行用空格或英文分号分隔，路径含空格或英文分号时，请使用英文引号包裹路径。
- 所有交互位置中，`00` 表示退出脚本，`0` 只表示返回或跳过，不再用于退出脚本。
- 交互路径输入阶段，`0` 返回模式菜单，`00` 直接退出脚本；返回菜单时会放弃本次已输入但未执行的路径。
- 交互模式下，每轮扫描和删除流程完成后会回到扫描模式菜单；带路径参数运行时只执行一轮，便于命令行批处理。
- 所有模式都会自动去重重复输入的目录，并拒绝父子目录，避免同一文件在一次运行中被重复扫描或重复处理。
- 除 `-yes` 外，删除前会先显示详细预览；预览和删除摘要会显示计划删除数量以及预计可释放空间。`-yes` 会输出汇总后进入倒计时；没有可删除项时会直接跳过。单目录和多目录合并模式提供 `默认删除 / 手动删除 / 跳过本次操作 / 退出脚本`，参考目录模式提供 `默认删除 / 跳过本次操作 / 退出脚本`。多个单目录逐个操作时，`0` 表示跳过当前目录，`00` 表示退出脚本。
- 默认保留规则：同一目录内优先保留文件名更短的文件；不同目录之间优先保留父目录中的文件；不属于父子目录时，优先保留目录层级更少的文件；目录层级相同时，优先保留目录路径更短的文件；仍相同时按文件名和完整路径排序。
- 删除会直接移除文件，不会进入回收站；扫描、哈希或删除失败时会提示并继续处理后续文件。
- 仓库 `Share\Remove-DuplicateFiles.ps1` 是 PowerShell 5.1 兼容共享版，保留完整单文件实现；允许脚本执行可运行 `Set-ExecutionPolicy -Scope LocalMachine -ExecutionPolicy RemoteSigned`。

### 哈希策略

脚本先比较文件大小，再比较部分 SHA-256，最后只对候选文件计算完整 SHA-256。部分哈希默认读取文件头尾各 64KB；文件不超过两个采样片段总长时会直接按完整文件计算部分哈希，后续不再重复计算完整哈希。候选文件的部分哈希和完整哈希默认使用 8 个并发 worker，每个 worker 会批量处理文件以减少 runspace 调度开销；可修改脚本顶部 `$HashParallelThrottleLimit` 调整，设为 `1` 可退回串行。普通扫描会用内存哈希表保存文件大小、部分哈希和完整哈希的小桶，减少大批量管道分组开销。参考目录模式会先建立“文件大小 -> 参考文件列表”的轻量索引；部分哈希和完整哈希都按需懒加载、批量并行计算并在本次运行内缓存，后续目标目录命中同一参考文件时不会重复计算。

## Remove-EmptyFolders.ps1

`Remove-EmptyFolders.ps1` 用于删除一个或多个指定文件夹下的空子文件夹。

脚本需要 PowerShell 7。空文件夹指目录内没有任何子项；隐藏项和系统项也会参与空目录判断。脚本只删除输入目录下的子文件夹，不删除输入目录本身。

### 参数说明

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `Path` | 无 | 一个或多个文件夹绝对路径；未输入路径时会引导交互输入。 |
| `-s` | 关闭 | 把隐藏文件夹也作为删除候选。默认不处理隐藏文件夹。 |
| `-yes` | 关闭 | 跳过详细预览和菜单，输出汇总后等待 10 秒，再执行默认删除；倒计时期间可按 `Enter` 取消。 |
| `-h` | 关闭 | 显示帮助信息。 |

### 使用示例

查看帮助信息：

```powershell
pwsh -File .\Remove-EmptyFolders.ps1 -h
```

扫描并删除一个目录下的空子文件夹：

```powershell
pwsh -File .\Remove-EmptyFolders.ps1 "C:\Path\Folder"
```

扫描多个目录：

```powershell
pwsh -File .\Remove-EmptyFolders.ps1 "C:\Path\A" "C:\Path\B"
```

包含隐藏文件夹作为删除候选：

```powershell
pwsh -File .\Remove-EmptyFolders.ps1 -s "C:\Path\Folder"
```

跳过详细预览，输出汇总后执行默认删除：

```powershell
pwsh -File .\Remove-EmptyFolders.ps1 -yes "C:\Path\Folder"
```

### 注意事项

- 路径必须是 Windows 文件夹绝对路径；命令行传参时，路径包含空格、括号等 PowerShell 特殊字符，请使用英文引号包裹路径；交互输入可分行，也可在同一行用空格或英文分号分隔，路径含空格或英文分号时，请使用英文引号包裹路径。
- 输入目录会按真实路径自动去重，但不能互为父子目录，避免输入根目录被另一个父目录扫描后当作子目录处理。
- 删除前会先显示详细预览；可选择默认删除、手动删除或退出脚本。
- 默认会从深到浅计算删除计划，因此删除子空目录后新变空的父目录也会在同一轮删除。
- 空文件夹判断始终会检查隐藏项和系统项；如果目录中存在隐藏文件或隐藏子目录，则不会被误判为空。
- 符号链接、联接点等重解析点目录不会作为删除候选。
- 删除会直接移除空文件夹，不会进入回收站；如果普通路径检查或删除失败，会用扩展路径 `\\?\` 重新确认目录为空后再兜底删除；扫描、判断或删除失败时会提示并继续处理后续目录。
